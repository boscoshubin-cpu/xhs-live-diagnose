# 小红书直播数据分析 Skill

从小红书后台直播数据截图出发，做系统性的直播表现诊断和优化建议。

## 前置条件

**用户需提供**：小红书直播后台数据截图（ark.xiaohongshu.com），至少包含：
- 直播概况（观看人数、成交金额、成交人数、客单价）
- 观众画像（性别、年龄、消费力、城市等级）
- 流量来源（推荐流、笔记引流、预约进入、回访）

## 四维诊断框架

### 维度 1：基础数据总览

| 指标 | 示例 | 健康基准 |
|------|------|---------|
| 观看人数 | 253 | <500=偏低；500-2000=中等；2000+=优秀 |
| 成交金额 | ¥722 | 按品类差异，首场有转化即合格 |
| 成交人数 | 3 | >观看人数1%为正常 |
| 客单价 | ¥144 | 总成交÷成交人数，评估定价策略 |
| 平均观看时长 | 2m34s | <2min=差；2-5min=一般；5min+=优秀 |
| 互动人数 | 12 | 低=缺少互动引导 |
| 新增粉丝 | 5 | 转化率=新增÷观看，>3%为优秀 |
| 预约人数 | 0 | =0说明未做预热，是最大流量缺口 |
| 退款金额 | ¥0 | 低=产品质量过关 |
| 观看-成交率 | 1.19% | >1%为正常 |

### 维度 2：观众画像

| 维度 | 分析要点 |
|------|---------|
| 性别 | 珠宝/美妆类93%+女性为正常 |
| 年龄 | 主力年龄段决定话术风格（25-30偏年轻/31-35偏品质） |
| 消费力 | >50%高消费力=中高客单价可行 |
| 城市等级 | 一二线主导=受众有购买力 |
| 人群标签 | 宝妈/白领/学生，不同标签不同打法 |

### 维度 3：流量来源结构

- **推荐流占比**：自然推荐比例
- **笔记引流**：0=直播前未发预告笔记
- **预约进入**：0=未做预热
- **回访观众**：低=没有固定开播节奏

**诊断逻辑**：
- 推荐流>80%但总观看低 → 平台给了流量但没留住人
- 笔记引流=0 → 最常见的流量问题，没做播前预热
- 回访=0 → 没固定开播时间，用户没养成习惯

### 维度 4：漏斗分析

```
曝光 → 进入 → 互动 → 商品点击 → 成交
```

| 环节 | 健康基准 | 偏低信号 | 优秀信号 |
|------|---------|---------|---------|
| 互动率 | ≥10% | <5%→平台判定内容差→减少推流 | >15%→增加推流 |
| 关注率 | ≥5% | 内容/人设弱 | 有吸引力 |
| 商品点击率 | ≥8% | 话术未引导 | 展示有效 |
| 转化率 | ≥2% | 价格/信任问题 | 商品+话术到位 |
| 平均观看时长 | ≥2min | 内容留不住人 | 有可看性 |

## 流量竞争分析

| 推荐流占比 | 诊断 |
|-----------|------|
| >70% | 平台还在给流量，问题是留存/互动 |
| 40-70% | 推流在减弱，检查互动率 |
| 20-40% | 推流明显减少，检查有无违规/水印问题 |
| <20% | 大直播间抢流量，换时段或加预算 |

## 新手扶持期

新账号有**3-7场临时流量扶持**：
- 第4场开始流量下降 ≠ 账号出问题，是扶持结束
- 此时要从「吃扶持」切换为「可持续运营」

## 互动率 — 核心指标

| 互动率 | 平台反应 |
|--------|---------|
| <5% | 判定内容无趣 → 减少推流 |
| 5-10% | 正常推流 |
| >10% | 增加推流 |

**提升互动**：
1. 开场3分钟不卖货，先提问：「姐妹们你们戴银饰会过敏吗？」
2. 福袋：「扣1抽小礼物」，设「关注+评论」条件
3. 每5分钟：「刚进来的宝宝扣个1」「想要这款的扣想要」
4. 目标：互动率 >10%

## 商品结构分析

| 价格带 | 角色 |
|--------|------|
| ¥0.9-5 | 福利款 — 留人+拉互动 |
| ¥29-50 | 入门款 — 降低首单门槛 |
| ¥100-300 | 利润款 — 主力成交 |
| ¥500-800+ | 高端款 — 拉高客单价 |

## 常见问题速查

- **投薯条后数据反而差？** — 可能人群不匹配。笔记内容和直播间卖的不一致会污染数据。停投，恢复2-3场自然流量后重新校准。
- **高流量后突然暴跌？** — 大概率是新手扶持结束。解法：固定开播节奏+预告笔记+每场上新。
- **高客单价反而好卖？** — 说明观众消费力强，不要降价，走品质路线。

## 交互工具：直播数据诊断 HTML

当用户（或其女票）需要可视化填表诊断时，使用 `templates/直播数据诊断.html`。

**文件位置**：`~/Desktop/hermes/直播数据诊断.html`（也存于本 skill 的 `templates/`）

**页面结构（上传优先）**：
1. 顶部 — 📷 拖拽/点击上传区（支持 PNG/JPG/HEIC）
2. 上传后 — 截图预览（带「换图」「移除」按钮）
3. 下方自动展开 — 三区表单：基础数据 / 流量来源 / 观众画像
4. 底部 — 「🔍 开始诊断」按钮
5. 结果区 — 5个指标卡 + 四段诊断 + 新手提醒 + 优化建议

**给用户的使用流程**：
1. 女票手机上截图小红书直播数据后台
2. 电脑打开 HTML → 拖入截图（上方预览对照，下方填表）
3. 对照预览填完数字 → 点「开始诊断」
4. 纯本地运行，无数据上传 🔒

**从 Hermes 发送给用户**：
1. `write_file` 生成 `~/Desktop/hermes/直播数据诊断.html`（从 skill `templates/` 复制）
2. `send_message` 发送文件：`MEDIA:/Users/yanxi/Desktop/hermes/直播数据诊断.html`

**展示 UI 截图给用户**：`browser_navigate` 打开文件 → `browser_vision` 截图。

**⚠️ 浏览器沙箱坑**：`browser_console` 直接调 `analyze()` 可能读不到 input value（所有值为 0）。两种正确方式：
- 让用户自己填数据点按钮（推荐）
- 或用 `browser_type` 逐字段填入后再调 `browser_click` 点按钮
